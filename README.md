# 🍽️ Swiggy SQL Analysis

A SQL-based data analysis project on Swiggy restaurant data. This project explores restaurant ratings, costs, cuisines, city-wise performance, and customer engagement metrics using MySQL queries to generate meaningful business insights.

## 📊 Project Overview

The objective of this project is to analyze Swiggy restaurant data and answer real-world business questions using SQL. The analysis focuses on:

- Restaurant ratings and ranking
- Cost analysis
- Cuisine distribution
- City-wise restaurant performance
- Rating count trends
- Aggregate and grouped insights

## 🛠️ Tools Used

- MySQL
- SQL
- Git & GitHub

## 📌 Key SQL Concepts Covered

- SELECT Statements
- Filtering with WHERE
- ORDER BY
- LIMIT
- Aggregate Functions (COUNT, AVG, MIN, MAX)
- GROUP BY
- HAVING
- DISTINCT
- IN & BETWEEN Operators
- Logical Operators (AND, OR)

## 🔍 Business Questions Solved

- Top-rated restaurants in the dataset
- Restaurants with rating counts above a threshold
- Total number of restaurants
- Average restaurant cost
- Cheapest restaurants available
- Minimum and maximum cost by cuisine
- Average ratings by city
- Number of unique cities
- High-performing restaurants based on ratings and reviews
- Cities with multiple cuisine options
- Cuisines with the highest restaurant counts

## 📈 Sample Insights

- Identified the highest-rated restaurants.
- Compared average ratings across cities.
- Analyzed cuisine-wise pricing patterns.
- Evaluated restaurant popularity using rating counts.
- Explored city-wise food diversity through cuisine availability.

## 🚀 How to Run

1. Import the Swiggy dataset into MySQL.
2. Create and select the database:

```sql
USE swiggy;
```

3. Execute the queries from `Swiggy.sql`.
4. Analyze the results and derive insights.
